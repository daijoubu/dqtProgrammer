<objective>
Add ZIP archive support to firmware loading, enabling the tool to extract .bin files from ZIP archives for the reprogramming process.
</objective>

<context>
@src/firmware.py

**Existing functionality (preserve):** FirmwareLoader.load() already handles .bin files with validation, CRC32 calculation, and size checking.

**Phase requirement:** Add support for loading firmware from ZIP archives while maintaining backward compatibility with existing .bin file loading.
</context>

<tasks>

<task type="auto">
  <name>Task 1: Add ZIP extraction support to FirmwareLoader</name>
  <files>src/firmware.py</files>
  <action>
Add ZIP file support to FirmwareLoader class:

1. Import `zipfile` module at top of file
2. Add new method `load_from_zip(self, zip_path: str) -> FirmwareInfo`:
   - Open ZIP file with zipfile.ZipFile (handle BadZipFile exception)
   - Find all .bin files (case-insensitive search using .lower().endswith('.bin'))
   - If no .bin found → raise FirmwareError("No .bin file found in ZIP archive")
   - If multiple .bin files → use first one found (log a warning)
   - Extract to temp directory using ZipFile.extract()
   - Load the extracted .bin file using existing load() method
   - Clean up temp extraction after loading (or use context manager)
3. Update `load()` method to detect .zip extension and route to load_from_zip():
   - Check file_path.lower().endswith('.zip') → call load_from_zip()
   - Otherwise → existing .bin loading logic
4. Update FirmwareError to include specific error types for ZIP-specific failures
  </action>
  <verify>
    <manual>Test with a ZIP containing a .bin file, verify extraction and loading works</manual>
  </verify>
  <done>
    - FirmwareLoader.load('firmware.zip') extracts and loads .bin correctly
    - Existing .bin loading continues to work unchanged
  </done>
</task>
